# Java Reversi AI

大学の授業課題として作成した、「αβ法と局面評価関数を用いたJavaリバーシAI」です。
探索アルゴリズムと評価関数の設計を通じて、局面に応じた意思決定を行うAIの実装を目的としました。

## 概要

本AIでは、以下の方針で設計を行いました。

- αβ法による探索を用いて最善手を探索
- 局面評価関数を「複数特徴量 × 重み」の線形和として定義
- ゲーム進行度（序盤・中盤・終盤）に応じて評価関数の重みを動的に切り替え

これにより、局面に即した柔軟な判断を行うAIを目指しました。


## アルゴリズム

### αβ枝刈り
Minimax法にαβ枝刈りを組み合わせることで、探索の効率化と計算時間の削減を行っています。
また、探索候補手の順序を一部ランダム化することで、探索順序の偏りを抑える工夫を行いました。

## 局面評価

局面評価には、以下の6つの特徴量を用いています。

- 石の数差
- 角
- 角周囲
- 辺（Edge）
- 着手可能数
- 安定石（隅から連続して確定した石）

特に 「安定石」を明示的に評価対象とすることで、終盤における安定した勝利を狙う設計としています。

## 動的重みづけ

盤上の石数合計をもとにゲーム進行度を判定し、評価関数の重みを切り替えています。

- 序盤：角の確保・可動域を重視  
- 中盤：各特徴量をバランス良く評価  
- 終盤：石数差・安定石を重視  

これにより、局面ごとの戦略的判断を可能にしました。

## 実験結果 / 考察
- 評価関数の特徴量設計
- 動的重み付け
- αβ法による効率的探索

以上の3点を軸とした設計により、終盤で安定した展開を作りやすいAIを実装できました。

今後の課題としては、
- 特徴量の重みの最適化
- 安定石判定のさらなる精密化
などが挙げられます。
